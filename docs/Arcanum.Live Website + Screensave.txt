# Arcanum.Live Website + Screensaver System Reference Brief

## Purpose

Arcanum.Live should be built as a premium immersive entertainment platform, not just a portfolio site. It should function as:

* A cinematic storefront for 360 movie rentals
* A dome show licensing portal
* A mythic AI film/world hub
* A dynamic screensaver / visualizer experience
* A future home for prompting tools and interactive galleries
* A marketing engine that pushes viewers to Vimeo On Demand, licensing inquiries, and store links

Core positioning:

**Enter the MythMachine**

**Rent 360 Movies for VR Headsets**

**Immersive Dome Shows**

**AI Cinematic Worlds**

**The Arcanum MythMachine**

---

# 1. Best overall tech direction

## Recommended stack

Use:

* GoDaddy for domain/DNS only
* Vercel for hosting
* Next.js / React for the site and app
* TypeScript
* Tailwind CSS
* Framer Motion
* Three.js / React Three Fiber for advanced visuals
* Web Audio API for music-reactive visuals
* Supabase for backend, database, auth, storage, playlists, and admin data
* Vimeo On Demand for paid 360 movie rentals
* Etsy/Shopify/Fourthwall or external store links for commerce at first

## Best final recommendation

**GoDaddy domain → Vercel hosting → Claude Code / Codex-built Next.js site → Vimeo On Demand embeds/CTAs → Supabase backend → custom WebGL screensaver page → external Etsy/store links.**

This is the best balance of:

* Low cost
* Ownership of code
* Dynamic backend
* Advanced screensaver freedom
* No no-code lock-in
* Future scalability

## Avoid

Avoid building the entire system in:

* Hostinger AI Builder
* GoDaddy shared hosting
* WordPress plugin chaos
* A closed no-code site builder

Those can work for simple brochure pages, but not for the complex screensaver/music-reactive/playlist/gallery engine.

# 10. Screensaver system concept

The screensaver should be a dedicated web app route:

```txt
/screensaver
```

It should be something people can leave running on a site, TV, browser, projection, or ambient display.

Core concept:

A fullscreen cinematic visual system that lets users choose an IP/world, a visual mode, and a music playlist. It then plays images, videos, music, particles, and audio-reactive effects.

## Main screensaver page controls

The page should have:

3. IP/world selector cards
4. Music playlist selector
5. Visual playlist selector
6. Fullscreen button
7. Shuffle button
8. Mute/unmute
9. Volume slider
10. Intensity slider
11. Speed slider
12. Particle amount slider
13. Bloom/glow slider
14. Next / previous buttons

# 11. Screensaver modes

## Mode 1: Gallery Drift

Purpose: turn uploaded images into animated cinematic wall art.

Features:

* Pulls images from selected playlist
* Slow zoom/pan
* Crossfade transitions
* Parallax movement
* Optional floating particles
* Optional text title overlay
* Optional IP logo overlay
* Fullscreen mode

Start with CSS transforms and Framer Motion. Add WebGL later.

## Mode 2: Video Temple

Purpose: play looping cinematic video clips.

Features:

* Pulls videos from selected playlist
* Autoplay muted by default
* Crossfade between videos
* Optional music playlist over video
* Optional overlay particles
* Fullscreen mode
* Video poster fallback

## Mode 3: Music Reactor

Purpose: music-driven particle visualizer.

Features:

* Pulls tracks from selected playlist or album
* Uses Web Audio API analyzer
* Bass controls particle expansion
* Mids control rotation/swirl
* Highs control sparkle/flicker
* Background image/video can be selected from IP world
* Three.js particle field
* Fullscreen mode

## Mode 4: Fluid Oracle

Purpose: premium audio-reactive fluid/energy simulation.

Features:

* WebGL canvas
* Audio-reactive color waves
* Mouse/touch interaction
* Bass creates ripples
* High frequencies create light flecks
* Performance settings: Low, Medium, High, Ultra

Start with a simpler shader or particle displacement. Do not overbuild fluid simulation in V1.

## Mode 5: MythMachine Shuffle

Purpose: automatic mixed-mode screensaver.

Features:

* Randomly rotates through images, videos, particles, audio, IP worlds
* User can select one IP, all IPs, high energy, chill, cosmic, ritual, festival, dome visuals, etc.
* Should feel alive but not chaotic.

# 13. Dynamic content behavior

The system should be upload-driven.

When a new IP world is added:

* It appears on /worlds
* It generates /worlds/[slug]
* It becomes available as a screensaver filter
* It becomes available for media tagging

When new images are added:

* They appear in the media library
* If assigned to a playlist, they appear in that gallery/screensaver playlist
* If assigned to an IP world, they appear on that world page

When new music is added:

* Tracks appear under /music
* If assigned to an album, they appear in that album
* If assigned to a screensaver playlist, they appear in the music selector

When a new screensaver preset is added:

* It appears as a launchable card/button on /screensaver
* It uses assigned playlist and visual settings

When a new Vimeo film is added:

* It appears on /films
* It generates /films/[slug]

# 18. Performance requirements

Screensaver must be beautiful but not melt laptops.

Add performance settings:

```txt
Low
Medium
High
Ultra
```

Low:

* CSS animation
* Fewer particles
* No heavy fluid simulation

Medium:

* Basic Three.js particles
* Moderate bloom

High:

* More particles
* Audio-reactive effects

Ultra:

* More expensive shader/fluid visuals
* Better bloom
* Higher particle count

Also:

* Pause animations when tab is hidden
* Lazy-load videos/images
* Use optimized thumbnails
* Do not autoplay audio until user clicks
* Fullscreen mode must work after user interaction
* Preload only current and next video

---

# 19. Media handling rules

## Images

* Use thumbnails in grids
* Use full image only in screensaver/player
* Store alt text where possible

## Videos

* Use poster images
* Do not load all videos at once
* Preload only current and next video
* Allow muted autoplay

## Music

* Require user click before playback
* Show track title
* Show album art
* Support next/previous
* Audio analyzer reacts only after playback starts

# 24. Song extractor tool idea

Before importing hundreds/thousands of songs into Arcanum.Live, build a separate local extractor app.

Purpose:

**ChatGPT export zip → find song conversations → extract songs → review/edit → export JSON/CSV → upload to Supabase/Claude/NotebookLM**

Recommended tool:

* Codex or Claude Code
* Local Next.js app
* No cloud upload required in V1

Extractor app name:

```txt
Arcanum Song Extractor
```

Input:

```txt
chatgpt-export.zip
conversations.json
```

Outputs:

```txt
songs_master.json
songs_master.csv
albums.csv
ip_worlds.csv
music_types.csv
screensaver_playlists.csv
```

Important: JSON should be master format because lyrics and Suno prompts break CSV easily.

Song schema:

```txt
title
slug
album
ip_world
music_type
mood
energy_level
bpm
vocal_style
instrumentation
fx_tags
visual_pairing
is_adult_adjacent
is_family_safe
audio_url
cover_image
lyrics
suno_style_prompt
version_type
date_created
source_chat_title
source_conversation_id
source_message_id
source_excerpt
title_confidence
extraction_confidence
```

Extractor UI:

1. Upload screen
2. Scan progress screen
3. Song-related conversation list
4. Extracted song table
5. Song detail editor
6. Duplicate review screen
7. Filters by ip_world, album, music_type, adult/family flag, version_type, confidence
8. Export screen

Workflow:

```txt
1. Export ChatGPT data
2. Download export zip
3. Run local extractor app
4. App scans conversations.json
5. App identifies song-related conversations
6. App splits huge chats into manageable chunks
7. App extracts song records
8. User reviews/edits inside app
9. App exports JSON/CSV
10. Import cleaned data into Supabase
11. Upload clean catalog to Claude Project / NotebookLM / ChatGPT files
```

```txt
Build a local-first Next.js App Router application called Arcanum Song Extractor.

Purpose:
The app lets me upload my ChatGPT data export zip or conversations.json file, scans the conversations, identifies song-related chats, extracts every song, remix, draft, variant, Suno style prompt, lyric block, and album concept, then lets me review/edit the extracted catalog and export JSON/CSV files for Supabase and Arcanum.Live.

Tech stack:
- Next.js App Router
- TypeScript
- Tailwind CSS
- React
- Local file processing in browser when possible
- If needed, use a local Node API route for parsing large JSON files
- No cloud upload required for V1
- Store working extraction data in IndexedDB or localStorage for V1
- Export downloadable JSON and CSV files

Core input:
- chatgpt-export.zip
- conversations.json

Core outputs:
- songs_master.json
- songs_master.csv
- albums.csv
- ip_worlds.csv
- music_types.csv
- screensaver_playlists.csv

Song schema:
title
slug
album
ip_world
music_type
mood
energy_level
bpm
vocal_style
instrumentation
fx_tags
visual_pairing
is_adult_adjacent
is_family_safe
audio_url
cover_image
lyrics
suno_style_prompt
version_type
date_created
source_chat_title
source_conversation_id
source_message_id
source_excerpt
title_confidence
extraction_confidence

Extraction rules:
- Do not invent songs.
- Extract only content present in the uploaded conversations.
- Detect Suno lyrics using section headers like [Intro], [Verse], [Pre-Chorus], [Chorus], [Bridge], [Final Chorus], [Outro].
- Detect style prompts using words like style, genre, instrumentation, FX, vocal, bpm, Suno, chorus, remix, downtempo, psytrance, anthem, cinematic, trance, ambient.
- Keep remixes and meaningful variants as separate records.
- Mark exact duplicates as duplicates.
- Preserve Suno formatting exactly.
- Preserve lyrics exactly.
- If a title is missing, create a provisional title from the strongest repeated phrase and set title_confidence to low.
- Mark HoneyVerse, Yum Yum, Nyx Arcana, Fanvue, adult/sensual material as is_adult_adjacent=true and is_family_safe=false.
- Mark general Arcanum, dome, mythology, goddess, cosmic, AI, and instrumental material as family_safe=true unless clearly adult-adjacent.

UI requirements:
1. Upload screen
2. Scan progress screen
3. Song-related conversation list
4. Extracted song table
5. Song detail editor
6. Duplicate review screen
7. Filters by ip_world, album, music_type, adult/family flag, version_type, confidence
8. Export screen

Important:
For V1, create a deterministic parser first using regex and heuristics. Do not require an LLM API key at first. Add optional LLM enrichment later.
```

# 30. Homepage Hero Spec v1: The Arcanum Portal Entrance

## Purpose

The homepage hero is not a standard website banner. It is the first chamber of the Arcanum metaverse.

It should feel like the visitor is standing at a dimensional threshold, with the whole site presented as a living map of worlds, media portals, archives, dome shows, and interactive experiences.

Core feeling:

```txt
You are entering The Arcanum.
The homepage is a portal chamber.
The image is the map.
The buttons are gateways.
The cursor awakens the world.
The scroll pulls you deeper.
```

## Cursor parallax behavior

Use mouse or pointer movement to create subtle cinematic depth.

Rules:

```txt
background moves least
foreground pillars move moderately
guardian character moves slightly more
floating glyphs move most
buttons move very little for readability
portal ring rotates independently
```

Recommended movement range:

```txt
background: 4-8px
midground: 8-16px
foreground: 12-24px
particles/glyphs: 20-40px
```

Desktop behavior:

```txt
cursor position controls parallax offset
hovering hotspots intensifies the related area
```

Mobile behavior:

```txt
use device tilt only if reliable, otherwise use slow idle parallax
no required hover interactions
buttons remain visible and tappable
```

Accessibility/performance:

```txt
respect prefers-reduced-motion
allow motion intensity setting later
keep parallax subtle to avoid nausea
```

```txt

foreground layers separate
floating glyphs accelerate slightly
```

At mid-scroll:

```txt
camera pushes toward central portal
portal glow brightens
rings rotate faster
world darkens around edges
short text appears: Choose Your Path or Step Deeper Into The Arcanum
```

After pinned hero sequence:

```txt
hero unpins
page scrolls into next immersive panel
next section begins with Dome Shows or 360 Movies
```

Optional:

```txt
clicking central portal can jump to /screensaver or open a Choose Your Path overlay
```
